# 🌐 Webgrafía — Semana 08: Arrays

## Referencias Principales

### 1. MDN Web Docs — Array

**URL**: https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/Array
**Idioma**: Español disponible
**Por qué usarlo**: Referencia oficial completa de todos los métodos de Array. Usar como diccionario cuando necesitas saber qué parámetros acepta un método, qué retorna y qué soporte de navegadores tiene.

---

### 2. JavaScript.info — Arrays

**URL**: https://javascript.info/array
**Idioma**: Inglés
**Por qué usarlo**: Tutorial interactivo con ejemplos ejecutables en el navegador. Excelente para ver código en acción sin necesidad de instalar nada.

---

### 3. JavaScript.info — Array Methods

**URL**: https://javascript.info/array-methods
**Idioma**: Inglés
**Por qué usarlo**: Referencia completa de métodos funcionales (`map`, `filter`, `find`, `reduce`). Incluye diagramas visuales y ejercicios con soluciones.

---

### 4. JavaScript.info — Spread Syntax

**URL**: https://javascript.info/rest-parameters-spread
**Idioma**: Inglés
**Por qué usarlo**: Explica la diferencia entre rest parameters y spread operator, con ejemplos claros de cuándo usar cada uno.

---

### 5. Python Tutor (JavaScript) — Visualizador de código

**URL**: https://pythontutor.com/javascript.html
**Idioma**: Inglés (la herramienta es visual)
**Por qué usarlo**: Herramienta visual que permite ejecutar código paso a paso y ver cómo los arrays cambian en memoria. Especialmente útil para entender:

- Cómo funciona la referencia vs copia con spread
- Qué hace `splice` exactamente
- Cómo itera `forEach` y `map`

> 💡 **Truco**: Copia cualquier fragmento de código de los ejercicios en Python Tutor y presiona "Visualize" para ver la ejecución paso a paso.

---

### 6. Can I Use — Array methods

**URL**: https://caniuse.com/
**Por qué usarlo**: Para verificar el soporte de métodos modernos (como `.at()` de ES2022) en los navegadores más usados. Útil cuando necesitas saber si un método funciona en todos los navegadores.

---

> 💡 **Recomendación de flujo**: Usa MDN como referencia cuando tengas dudas específicas, JavaScript.info para aprender conceptos nuevos, y Python Tutor para depurar y entender qué hace tu código.
